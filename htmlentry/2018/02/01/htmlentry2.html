HTML入門[第二回]タグを理解する

access_time 2018年02月01日

HTMLはすべてHTMLタグを使って記述されています

インターネットで見るすべてのホームページはhtmlで記述されています。phpなどのプログラムで作られているホームページも最終的にはhtmlをプログラムが出力しているだけで、最終的にはすべてhtmlになります。今見ているページもすべてhtmlで記述されております。ブラウザでは実際のhtmlがどのようなものか見れるので見てみましょう。
※html文書に限らずプログラムが書かれたものをソース（ソースコード）と呼びます。

HTMLタグを見る

html文書を見るとすべての記述が記号の小なり「<」と記号の大なり「>」で挟まれた文字で記述されています。この小なりと大なりで囲まれた文字をタグと呼びます。
タグの記述の仕方は3種類あり

  • <タグ名>
    タグのみで意味を成すもの
  • <タグ名>要素</タグ名>
    タグとタグ名にスラッシュを付けた終了タグで文書などを挟むもの。タグで挟まれた文書などを要素といいます
  • <タグ名 />
    一つのタグで最後に閉じる証としてスラッシュ「/」を記述するものxhtmlを記述する際に用います と以上の３つの種類があります。そしてをタイトルタグ<meta />をメタタグといいます。タグの中に書かれた文字＋タグというように〇〇タグと基本的には呼びます。

と以上の３つの種類があります。そして<title>をタイトルタグ<meta>をメタタグといいます。タグの中に書かれた文字＋タグというように〇〇タグと基本的には呼びます。

HTMLタグの属性

タグを見てみると、<head>などのようにタグ名だけが記述されているものもありますが、<meta charset=”UTF-8“>のようにcharaset=”UTF-8″などとタグ名以外のことも記述されているタグがたくさんあります。この記述のcharasetを属性と呼びUTF-8を属性値と呼びます。タグには様々な属性値を設定することができ見た目や動きなどを細かく設定するための機能です。

例えば<html lang=”ja”>とあります。<html>と</html>に挟まれた内側はすべてhtmlで記述されていますという意味のタグです。さらに属性でlang=”ja”とすることで中に書かれている文書は日本語で記述されていますという属性が設定されています。この設定がない場合に別の言語のパソコンで見たときに違う文字が表示される（いわゆる文字化けが起こる）可能性があります。

HTMLを書くということはこのタグと属性値を書くことがすべてです。すべてのタグや属性を覚える必要はありません、どのようなタグがあったかだけを覚えておけば調べながらでもHTMLをかけるようになります。しかしタグや属性などといった調べるとき必要な言葉は覚えてしまいましょう。

HTMLにはバージョンがあります

バージョンがあるということだけ覚えてください。現在公開されているホームページのほとんどすべては下記３つのバージョンいずれかのhtmlで記述されています。

  • html 4
  • xhtml
  • html 5

ほとんどのページはhtml5になってきているので基本的にはhtml4やxhtmlなどは理解する必要はありません。htmlは当初の目的とは少し違った発展をとげてきました。世界中のパソコンからブラウザさえあれば同じように表示される便利な文書として商用や趣味など多岐にわたる目的で使用されています。その発展に伴い新しいタグや不要となるタグが出てきてその都度新しいバージョンとしてhtmlが新しくなってきていまのhtml5に至ります。

htmlのバージョンは大きく動きが違うなどといったことはありません、どのタグが利用できてどのタグが利用できないという取り決めだけなのです。そして現在html5が勧告されてからだいぶ時間が立ちhtml5が処理できないブラウザはほぼサポート対象外になっているのでhtmlのバージョンを気にする必要はほぼありません。一部のWebサービスで重要なお客さんが古いPCでないとダメであるといったケースのみhtml4でも対応できる記述方法はありますがほぼ考えられません。

DOCTYPE宣言

htmlのバージョンはhtml冒頭に必ず記載する<!DOCTYPE>の部分で決まります。xhtmlのみDOCTYPE宣言前にxmlであることを宣言する必要があります。今後作るhtmlは特殊な事情や指定がない限りhtml5でよいです。

  • html5
    <!DOCTYPE html>
  • html4
    厳密型-W3c非推奨のタグや属性が使えません（見た目に関する属性など）
    <!DOCTYPE HTML PUBLIC “-//W3C//DTD HTML 4.01//EN” “http://www.w3.org/TR/html4/strict.dtd”>
  • html4
    移行型-w3c非推奨のタグや属性が使える
  • xhtml
    <!DOCTYPE HTML PUBLIC “-//W3C//DTD HTML 4.01 Transitional//EN” “http://www.w3.org/TR/html4/loose.dtd”>
    <?xml version=”1.0″ encoding=”UTF-8″?>
    <!DOCTYPE html PUBLIC “-//W3C//DTD XHTML 1.0 Strict//EN” “http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd”>
htmlのバージョンの中にはxhtmlという特殊なものもあります。これはhtmlはxmlとしても処理できればパソコンで処理がしやすく便利と考えて規定されたhtmlです。ところが基本的にタグを閉じないとエラーとなりすべて表示されないなどhtmlの敷居を少し上げてしまうためかほとんど普及しませんでした。膨大な数の他人のHTMLを解析するロボットの開発などにはメリットがあるかもしれませんが、htmlを書く側にとってはエラーが出るとすべて表示されないというデメリットしかなくxhtmlの定義に従ってhtmlを作るメリットがありませんでした。そのため発展はせず一部几帳面な人やプロジェクトのみで守られているようです。
