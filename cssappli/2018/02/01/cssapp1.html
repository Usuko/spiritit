【Layout全般】ブラウザ間で表示の差をなくす

access_time 2018年02月01日

ブラウザが違うと表示が違うことがよくあります

Webサイトを見るブラウザは現在たくさんの種類とそれぞれにバージョンがあります。自分が作成したWebサイトなどは見る人がどのブラウザ（IEかChromeか等）のどのバージョンを利用するかを選べません。そして、ブラウザ毎に表示の設定が違うのでそれぞれ見た目が異なることがよくあります。

ブラウザはそれぞれのデフォルトでスタイルシートを持っています。そのデフォルトのすたいーるシートをWebサイトなどで指定されたスタイルで上書きし表示しています。その際すべてのタグを上書きすれば問題はないのですが、基本的にすべてのタグを指定することは大変であるため上書きできされなかったタグがそれぞれのブラウザで違う表示になってしまうということが起こります。

簡略図ですがブラウザがスタイルを適用していく順番です。 それぞれ各ブラウザのデフォルトスタイルシートが適用され、そのスタイルをWebサイトのスタイルシートが上書きします。 最終的なスタイルシートで表示を行うためWebサイトで指定されていないスタイルはすべてブラウザ毎のデフォルトスタイルになります。そのデフォルトが違う場合に当然表示が異なる結果になります。

Webサイトをきちんと作成する場合には現状ほぼ全てのデザイナやプログラマはこのブラウザ間の表示を差をなくす処置をしています。（この処置をしてもまだブラウザ毎で違う表示になってしまう場合のものもありますが後述）

たいていは下記に説明するリセットをするかノーマライズをするかのどちらかをするのが主流です。

リセット

リセット.cssの考え方はブラウザがもともと採用しているスタイルシートのほとんどをその名のとおり「リセット」してしまう対応です。Webサイトの全てのタグなどをフルデザインする場合にはとても有効です。ただし開発にスピードが求められている場合やデザインにそれほど力を入れたくない場合などは後述する「ノーマライズ」のほうがむいているかもしれません。

有名なのは下記のようなリセットcssです

[Eric Meyer’s CSS reset]

[Yahoo! CSS Reset（YUI3）]

ノーマライズ

ノーマライズの考え方はリセットとは異なり、ブラウザごとのデフォルトスタイルシートの差を吸収し同じように表示されるよう標準化します。リセットの場合であると基本的にはデザインをまだしていないものは何もデザインされず表示されるのでtableやulやh1～h6など良く使うものもきっちりデザインしないといけません。

すべてをデザインする場合にはリセット、単純にブラウザ間で誤差をなくしたい場合にはノーマライズを使うのが正しいようです。

下記のサイトのcssが有名です [normalize.css]

normalize.cssを取り込んでみる

際にnormalize.cssを使ってノーマライズしてみます。

上記normalize.cssのページに行きダウンロードボタンを押します。

一部のブラウザではダウンロードは始まらず 直接cssのファイルが開く場合がありますのでその場合は全てを選択。

右クリックをしてコピーを選択

cssファイルを設置する場所に[normalize.css]という空のファイルを作成。 ダウンロードできた場合はnormalize.cssをここに移動またはコピーペーストします。

作成したファイルを開き、先ほどコピーした内容を貼り付けます。

メニューから[名前をつけて保存]を選びクリックします。

ダイアログが表示されるので[文字コード]でUTF-8を選び[保存ボタン]をクリックします。

normalizeを取り込むhtmlを開き。headerタグで今設置したcssへのリンクを張ります。 normalizeを取り込みたいhtmlを開きます。

linkタグでnormalize.cssへのリンクを張ります

<!DOCTYE html>
<html lang="ja">
	<head>
		<meta charset="UTF-8">

		<!--normalizeは他のcssの前に読み込みます-->
		<link rel="stylesheet" href="./css/normalize.css">
		<link rel="stylesheet" href="./css/style.css">
	</head>
	<body>
		test
	</body>
</html>

normalize.css確認方法

ノーマライズできているかどうかは若干分かりにくいです。その場合はh1タグをhtmlに記述し確認してください。 下記のようにh1タグをテストとして追記します。

<!DOCTYE html>
<html lang="ja">
	<head>
		<meta charset="UTF-8">
		<link rel="stylesheet" href="./css/normalize.css">
		<link rel="stylesheet" href="./css/style.css">
	</head>
	<body>
		<h1>test</h1>
	</body>
</html>

下記のように、文字がブラウザの一番左に詰めて表示されていればOKです

その他の方法としてCSS FrameWorkを利用

その他ブラウザ毎で表示の差が出ないようにするには、BootstrapなどのCSS FrameWorkと呼ばれるものを使うことでも対応できます。これはそのCSS FrameWorkが基本的にはリセットまたはノーマライズを内部的に行っているからです。

上記のようにしてWebサイトを作る際にブラウザ間での表示のズレを出来るだけなくすようにするのが標準的なWebサイトの作り方となっています。
