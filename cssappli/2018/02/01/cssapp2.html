【Layout全般】paddingとborderとmarginの計算

access_time 2018年02月01日

padding,border,marginまとめ

とてもごちゃごちゃになりやすく、すっきり理解していないと後々わからなくなってくるスタイルです。 padding,border,marginはとてもよく使う割に意外と大体で理解してる場合が大きくいったんすっきりまとめた方がよいです。

いっぺんに記述して違いを見てみる。 任意でファイルを作成し下記htmlを作り表示させます。

<!DOCTYPE html>
<html lang="ja">

	<head>
		<meta charaset="UTF-8">
		<style type="text/css">

			body{
				background-color:black;
				padding:0;
				margin:0;
			}

			.boxa{
				width:200px;
				height:100px;

				margin:15px 15px 15px 15px;
				padding:15px;
				border:15px solid red;

				background-color:blue;
			}

			.boxb{
				background-color:yellow;
				height:100%;
				width:100%;
			}

		</style>
	</head>


	<body>
		<div class="boxa">
			<div class="boxb">
				&nbsp;
			</div>
		</div>
	</body>


</html>

下記のような図形が表示されるのでそれぞれどれがどの部分かを把握しましょう。 ※文字は説明のため追記したものでhtmlでは表示されません。

margin

図を見てみると分かるように、bodyの背景を黒にしています。marginは指定したコンテンツ（boxa）の外側に余白がとられ黒が表示されます。

中身のコンテンツとそれ以外のコンテンツの間に余白を作りたい場合に指定するものです。

border

borderは文字通り、中身のコンテンツとそれ以外のコンテンツの境界線を表現したいときに指定します。

padding

paddingはコンテンツの境界とその内側に余白を作りたい場合にしていするものです。

サイズの計算

ここからが問題なのですが、このmargin、padding、borderはデフォルトの場合すべてコンテンツの幅などは加算されてしまいます。 一番内部の黄色い部分の幅はmargin,border,paddingの幅にかかわらず200pxで表示されます。つまりmargin,border,paddingを計算に含めて内部の幅を記述しないと表示が想定と違うことに鳴ります。

これはcssを使い始めた人が必ず混乱する箇所でもあるのでこのように計算されるということは覚えておいて損はありません。

ちなみに対処方法は下記のような方法があります。

計算する

横幅を指定するcssは常にこの計算を考慮に入れ指定する。
※sassを利用すると計算をしてくれるように設定できるのでsassを使っている場合は向いています。

box-sizingを使う

box-sizingというプロパティはその要素の計算方法を指定できます。 box-sizing:border-box;としてwidthを設定するとborderとpaddingのサイズを指定したwidthに収まるようプロパティを再計算してくれます。

先ほどの例を少しかえてborder-boxを指定したサンプルです。

<!DOCTYPE html>
<html lang="ja">

	<head>
		<meta charaset="UTF-8">
		<style type="text/css">

			body{
				background-color:black;
				padding:0;
				margin:0;
			}

			.boxa{
				width:200px;
				height:100px;

				margin:15px 15px 15px 15px;
				padding:15px;
				border:15px solid red;

				background-color:blue;

				box-sizing:border-box;
			}

			.boxb{
				background-color:yellow;
				height:100%;
				width:100%;
			}

		</style>
	</head>


	<body>
		<div class="boxa">
			<div class="boxb">
				&nbsp;
			</div>
		</div>
	</body>


</html>

上の図のようにwidthをborderとpaddingを含めたサイズで計算をしてくれます。marginは他との距離なのでこの計算には含みません。

図が重複しますが、box-sizingを利用しない場合下記のような計算になります。

幅を固定する

計算するの対処法と同じことですが、そのバリエーションとして。margin,padding,border固定してデザインすると計算は少し楽になります。

とにかく問題が起こるパターン

余白を取る際にあまり意識をせずmarginをつかったりpaddingを使うのを避けましょう。 その、一か所だけであればよいのですがその都度意識せずに余白を取るとWebサイトが大きくなってきたときにとてもごちゃごちゃして把握するのにとても時間がかかる場合があります。

決まった通り使うことを意識しておけばこういった後々の手間は減って行きます。
